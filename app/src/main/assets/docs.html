Hướng dẫn kết nối Webhook API

Tài liệu đầy đủ để tích hợp chuyển khoản tự động vào hệ thống QDev Sync của bạn.

✓ V7.5 Stable 2 Endpoints ⚡ XAMPP / PHP 7.4+

Mục lục

📌 Tổng quan 📱 Phương thức 1 — Android Forwarder App → Xác thực → Tham số gửi lên → Ví dụ thực tế 🏦 Phương thức 2 — SePay / Casso Webhook → Xác thực → Payload JSON → Ví dụ thực tế 🔑 Cú pháp nội dung chuyển khoản 📤 Định dạng Response ⚠️ Mã lỗi
📌 Tổng quan
QDev Sync hỗ trợ 2 phương thức nhận thông báo giao dịch. Chọn 1 trong 2 tuỳ thuộc vào hạ tầng của bạn.
📱
Android Forwarder
App chạy trên điện thoại, đọc thông báo push từ App ngân hàng và gửi POST đến server. Phù hợp khi không dùng VAS/API ngân hàng.
🏦
SePay / Casso
Dịch vụ đọc sao kê ngân hàng tự động, gọi webhook khi có giao dịch mới. Đáng tin cậy hơn, hỗ trợ nhiều ngân hàng.
⚠️
Chỉ cần cấu hình 1 trong 2 phương thức. Dùng cả 2 cùng lúc có thể gây nạp tiền trùng lặp.
📱 Phương thức 1 — Android Forwarder App
App đọc thông báo SMS/Push trên điện thoại Android và gửi POST về server của bạn.
POST https://yourdomain.com/appbank/webhook_private.php
🔑
Xác thực: Gửi token qua header X-Private-Token hoặc qua field token trong POST body.

Headers yêu cầu

HeaderGiá trịMô tả
X-Private-TokenstringToken cá nhân của tài khoản (lấy từ Tab API trong App)
Content-Typeapplication/x-www-form-urlencodedĐịnh dạng dữ liệu POST thông thường

POST Body

FieldKiểuBắt buộcMô tả
title string Tuỳ chọn Tiêu đề thông báo từ App ngân hàng
content string Bắt buộc Nội dung đầy đủ của thông báo. Phải chứa số tiền và mã NAPTIEN{ID}.
VD: MB: +50,000VND ND: NAPTIEN15 nap tien
token string Tuỳ chọn Có thể gửi token qua đây thay vì header

Ví dụ Request

# Gửi bằng cURL curl -X POST https://yourdomain.com/appbank/webhook_private.php \ -H "X-Private-Token: NAPTIEN_A3F2B9C1_15" \ -d "title=MB Bank&content=MB: +50,000VND tai MB ND: NAPTIEN15 nap tien"

Cách hệ thống xử lý nội dung

1
Trích xuất số tiền

Regex tìm pattern +50,000 hoặc nhận được 50.000đ trong nội dung.

2
Nhận diện người dùng

Regex tìm NAPTIEN\d+ trong nội dung. VD: NAPTIEN15 → User ID = 15.

3
Cộng số dư

Cập nhật balance trong bảng users và tạo bản ghi giao dịch.

4
Chống trùng lặp

Hash MD5 của nội dung được lưu để không xử lý cùng 1 giao dịch 2 lần.

🏦 Phương thức 2 — SePay / Casso Webhook
Dịch vụ đọc sao kê tự động gọi webhook khi có giao dịch mới vào tài khoản ngân hàng.
POST https://yourdomain.com/appbank/webhook.php
🔑
Xác thực: Header Authorization: Bearer {SECRET_TOKEN}. Lấy Secret Token từ Tab API trong App (Admin).

Headers yêu cầu

HeaderGiá trị
AuthorizationBearer qd_secret_webhook_778899
Content-Typeapplication/json

JSON Body (SePay format)

FieldKiểuBắt buộcMô tả
amountnumberBắt buộcSố tiền giao dịch (đơn vị: VND). VD: 50000
contentstringBắt buộcNội dung chuyển khoản. Phải chứa QD{ID}
referenceCodestringBắt buộcMã giao dịch duy nhất từ ngân hàng (dùng để chống trùng lặp)
gatewaystringTuỳ chọnTên ngân hàng. VD: MB, VCB
descriptionstringTuỳ chọnMô tả giao dịch (dùng thay content nếu không có)

Ví dụ Request (SePay gửi đến server)

POST /appbank/webhook.php HTTP/1.1 Authorization: Bearer qd_secret_webhook_778899 Content-Type: application/json { "amount": 50000, "content": "NAPTIEN15 nap tien thang 4", "referenceCode": "FT24115ABCDE", "gateway": "MB", "id": 12345 }
🔑 Cú pháp nội dung chuyển khoản
Để hệ thống tự động nhận diện người dùng, nội dung chuyển khoản phải chứa chuỗi NAPTIEN + ID tài khoản.
✅
Pattern nhận diện chính xác: NAPTIEN + khoảng trắng (tuỳ chọn) + số ID của người dùng.
Ví dụ: Tài khoản ID = 15 → ghi NAPTIEN15 hoặc NAPTIEN 15
# Regex được dùng trong cả 2 file PHP preg_match('/NAPTIEN\s*(\d+)/i', $content, $matches); # '\s*' = khoảng trắng tuỳ chọn | '/i' = không phân biệt hoa thường
Nội dung chuyển khoảnKết quả
NAPTIEN15 nap tien✅ Nhận diện → User ID 15
NAPTIEN 15✅ Nhận diện → User ID 15 (có khoảng trắng)
naptien15 thanh toan✅ Nhận diện → User ID 15 (viết thường)
THANH TOAN NAPTIEN15✅ Nhận diện → User ID 15 (ở cuối nội dung)
nap tien 50k❌ Bỏ qua — Không có chuỗi NAPTIEN
NAP15 nap❌ Bỏ qua — Thiếu chữ "TIEN"
⚠️
ID người dùng được hiển thị trong Tab API của ứng dụng. Ví dụ ID = 15 → nội dung chuyển khoản phải có NAPTIEN15.
📤 Định dạng Response

✅ Thành công

{ "status": "success", "message": "Balance updated successfully", "user_id": 15, "amount": 50000 }

⚠️ Bỏ qua (không nạp nhưng không lỗi)

{ "status": "ignore", "message": "Transaction content does not match pattern" }

❌ Lỗi

HTTP/1.1 401 Unauthorized { "status": "error", "message": "Unauthorized" } HTTP/1.1 400 Bad Request { "status": "error", "message": "Invalid JSON" }
⚠️ Mã lỗi & Xử lý
HTTP StatusNguyên nhânCách xử lý
200 OK Thành công hoặc bỏ qua (ignore) Kiểm tra field status
401 Sai Token xác thực Kiểm tra lại Token trong App và webhook_private.php
400 JSON không hợp lệ Kiểm tra Content-Type và định dạng JSON gửi lên
500 Lỗi server / DB Kiểm tra file bank_log.txt tại thư mục server
💡
Mọi request đến webhook_private.php đều được ghi vào file bank_log.txt để debug. Kiểm tra file này khi có vấn đề.
